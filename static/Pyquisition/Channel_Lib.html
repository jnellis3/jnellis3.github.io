Channel_Lib package

Abstract Channel and Display Widget

The Active Channel and Deactive Channel windows display channels that are based on the abstract Channel class. This class is designed to support the addition of arbitrary channel types to the system.

Functionality

The Channel class essentially serves as a node in a flowchart, containing internal inputs and outputs. During program execution, the process method is called for each channel. This method takes the current inputs, processes them, and updates the channel’s outputs. Channels can be interconnected by setting the outputs of one channel as the inputs of another channel.

Optional Control Widget

Channel types can optionally implement a control widget. When requested by the user, this widget is displayed in the MDI area, providing an interface for adjusting the channel’s settings.

class Channel_Lib.channel.Channel(channelName: str, configDict: Optional[dict] = None, parent=None)[source]

Bases: PyQt6.QtGui.QStandardItem

This is an abstract class for any channel that will be placed in the channelWidgets. The channel class is a subclass of QStandardItem, so it can be added to a QStandardItemModel.

Parameters

  • name – Name of the channel, this can be any string, but should be unique to the channel. This is used to identify the channel when it’s outputs are used as inputs to other channels or display widgets.

  • configDict – Dictionary containing the configuration of the channel. If present, the restoreState method will be called to restore the channel to the state it was in when the dictionary.

Variables

  • name – Name of the channel

  • inputs – Dictionary containing the inputs to the channel. Keys are the names of the inputs. The values are the input objects. The input objects can be any object, but should stay consistent across a subclass of Channel. For example, if a subclass of ‘InstrumentChannel’ is defined and listens to an ‘Instrument’ object as an input, then the input object should be an ‘Instrument’ object.

  • outputs – Dictionary containing the outputs of the channel. Keys are the names of the outputs. These values get updated after the process method is called. The output objects can then be used as inputs to other channels or display widgets.

  • children – List of channels that are children of this channel. This is used to keep track of the channels. In the case of an instrument channel, it’s children are going to be the output values of the instrument. If the channel is a mux channel however, it’s children are going to be the channels that are connected to the mux.

  • signals – This is a class that contains the signals that the channel can emit. Channels use these signals to inform the rest of the program when data should be logged, plots should be updated, etc.

class Channel_Lib.channel.ChannelSignals[source]

These are the signals relevent to channels. Channels should make sure to emit these signals when they have new data to be logged, plots to be updated, etc.

Variables

  • inputChanged – This signal should be emitted when the inputs to the channel have changed. The emitted object should be in the form of an array: [channelName:str, inputs:dict].

  • outputChanged – This signal should be emitted when the outputs of the channel have changed. The emitted object should be in the form of an array: [channelName:str, outputs:dict].

Instrument Read Channel

class Channel_Lib.Instrument_Read.InstrumentRead(channelName: str, state: Optional[dict] = None, parent=None)[source]

Bases: Channel_Lib.channel.Channel

The channel is the first implementation of the instrument queue.

This channel works similar to the older version the “instrument channel”, but instead of directly sending commands to the instrument, it requests that the reading be added to the instruments queue. This allows the instrument to be used by multiple channels without having to worry about the instrument sending conflicting responses.

Parameters

  • channelName (str) – The name of the channel

  • state (dict) – The state of the channel, if it is being restored from a saved state

  • parent (MainWindow) – The the program window that the channel is a part of. The channel needs to have reference to this because it needs to be able to access the instrument model manager
