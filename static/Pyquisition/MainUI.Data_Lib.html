class MainUI.Data_Lib.accumulator.Accumulator(file_location: str, config: Optional[dict] = None, parent=None)[source]

Bases: object

Object responsible for recording all outputs of channels displayed in the actChW. For the time being, this will only record in the long format. This means the file will need to be converted during post processing in order go to wide format.

Parameters

  • file_location (str) – Location of the file to record to

  • config (dict) – Dictionary containing the configuration of the channel. If present, the restoreState method will be called to restore the channel to the state it was in when config was created.

  • parent – The parent channel window (will likely always be actChW). This is used to get the channels displayed in the actChW.

  • autoUpdate – If true, the file name will change at midnight with the new date appended on

  • appendDate – If true, the date will be appended to the file name

  • timer – Timer used to call the process method

record_output(data: Optional[list] = None, fileName=None)[source]

This is the signal slot for when channels want to record their outputs. A channel will pass along “data”; which is a list containing the channel name, and a dictionary of outputs. They can also specify a filename if they are saving their outputs to a different file than the large datafile.

NOTE: The outputs in the dictionary could be a string, float, or int. The key of the dictionary is the name of the output.

NOTE: There will always be a key called “time” in the dictionary. This value is the timestamp generated by calling time.time()

Example of a channel passing along their packaged data:

data = ["Channel 1", {"Imag": 1, "Output 2": "Kelvin", "Output 3": 3.1415, "time": 123456789}]

Long format would convert this to something like:

Time: 342342354, Channel_name_Variable: 234234
